# Backend Architecture & Documentation

Welcome to the ACCESS Borrowing System & CMS backend documentation. The detailed planning documents have been broken down into specific files for easier reading:

- **[Product Requirements Document (PRD)](./backend/PRD.md)**: Target audience, core features, and development phases.
- **[Entity Relationship Diagram (ERD)](./backend/ERD.md)**: Database schema definitions including CMS, Assets, and Borrow Requests.
- **[Borrowing Flow & Asset States](./backend/BORROWING_FLOW.md)**: A visual state machine flowchart of the physical borrowing transaction.
- **[API Endpoints Specification](./backend/API_ENDPOINTS.md)**: Comprehensive mapping of all REST endpoints required.
- **[Best Practices & Security](./backend/BEST_PRACTICES.md)**: Architectural suggestions for authentication, file storage, and audit logging.
- **[Implementation Specs (Supabase)](./backend/IMPLEMENTATION_SPECS.md)**: A detailed, phase-by-phase development roadmap utilizing Supabase Auth, PostgreSQL, and Storage.
